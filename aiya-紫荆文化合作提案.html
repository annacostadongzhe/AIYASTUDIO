定位

AIYA 不是又一个 AI 工具

它是一个企业内部 AI 制片工作流系统。核心不是"能生成什么"，而是"能把专业制片判断交给系统流程"。

一句话定义

基于 GPT-Image 2 Edit + Dreamina / Seedance 2.0 的多 AGENT 企业内制片工作流系统。
核心价值：多 AGENT 制片认知封装——把导演、分镜、视觉、prompt 能力封装成流程，让组织级生产成为可能。

🎯

不是通用平台

不是无限画布、不是单AGENT prompt工具。它把专业制片判断封装成可确认的流程步骤。

🧠

认知封装

镜头语言、构图、表演、资产引用、质量门禁——这些专业能力被 AGENT 承担，不丢给操作员。

核心能力

多 AGENT 生产判断网络

8 个专业 AGENT 角色组成生产闭环，互相质检、迭代收敛，将专业制片流程系统化。

Story Director

负责故事结构、冲突推进、角色动机和叙事节奏，避免剧本只停留在原始描述。

Visual Director

负责画面语言、构图、色彩、空间关系和镜头可视化，把文字转成可执行的视觉判断。

Performance Director

负责表演、停顿、动作、情绪递进，让角色不是"站着说台词"。

Actor AGENT

带着角色身份参与讨论，补充人物反应、心理边界和表演细节。

HIDE

决策门禁和用户意图承接层，把确认、反驳、修正转成流程状态。

Storyboarder

把讨论结果拆成 SHOT，写入镜头目标、标注和下游生成要求。

Lighting

补足光线、氛围、空间层次，避免画面只有主体没有电影感。

Prompt Engineer

把资产、参考、镜头目标、质量约束封装成 SHOT 级 prompt。

关键区别

单 AGENT 是"你问它答"；AIYA 是多个制片角色互相质检，再把结果落到可生产的 SHOT。

商业价值

不是省按钮，是降低组织用人成本

商业价值评估的三个核心维度：岗位替代、培训周期、产能复制。

维度传统方案AIYA 方案
操作员要求高级复合人才：同时理解 prompt、分镜、构图、灯光、角色一致性、模型修复和视频生成低经验操作员 + 系统流程：主要负责任务确认、候选审核、补充修改
培训周期招聘稀缺复合人才，培训慢、复制难3 天上手内部工具培训，专业判断压缩为可确认步骤
产能可复制性依赖个人能力，难以批量复制多 AGENT + 资产体系 + 质量门禁，可以训练新人替代高级岗位
资产沉淀项目间资产难以复用，风格漂移角色/场景/道具资产系统化管理，跨项目一致性保障
失败返工依赖个人排查，效率低下自动补缺、候选重试、质量门禁闭环，不让流程断掉

差距不只是效率，而是组织可复制成本。一个系统化流程可以训练外人；一个高级复合人才很难批量复制。

— AIYA Studio System Evaluation 白皮书

对标参考

行业定位：不是画布工具，是生产系统

AIYA 的核心对标不是"谁有画布"，而是谁能把多 AGENT 制片认知封装成企业内部生产流程。

平台类型代表产品核心定位与 AIYA 差异
视觉协作容器Adobe Firefly Boards无限画布 + 视觉 brainstorming要会拆解、摆放、判断；只是创作容器，不替代多 AGENT 生产链
分镜故事板Boords / Katalist分镜生成和故事板协作仍需懂镜头与审美；偏单点辅助，不封装完整制片认知
视频生成工具Runway视频模型和创意生成强在模型，不等于企业内部工作流
脚本到视频LTX Studio脚本到视频的 AI Studio偏平台化操作，不是私有化生产系统
AIYA Studio本系统企业内部多 AGENT 制片工作流低经验操作员 + 审稿确认；把导演、视觉、表演、分镜、prompt、队列封装成流程
